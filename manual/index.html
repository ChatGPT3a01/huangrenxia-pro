第一章　安裝前準備

1.1　系統需求

在開始安裝之前，請確認您的電腦符合以下最低需求：

項目WindowsMac
作業系統Windows 10 以上（64 位元）macOS 12 (Monterey) 以上
處理器64 位元 x86 處理器Apple Silicon (M1+) 或 Intel
記憶體4 GB RAM 以上（建議 8 GB）
儲存空間500 MB 可用空間
網路需要網路連線（呼叫 AI API）
USBUSB 3.0 連接埠（建議直接插主機板）
v2.0 大幅簡化！

v2.0 採用原生 Node.js + OpenClaw 架構，不再需要 WSL、Docker、虛擬化或管理員權限。 安裝時間從 30 分鐘縮短至 1 分鐘以內，且同時支援 Windows 和 Mac。

1.2　安裝前確認清單

安裝前，請依序確認以下事項：

  • 電腦為 64 位元作業系統
  • 有穩定的網路連線（AI 服務需要網路）
  • 已準備好購買時綁定的 Email 地址
  • 已備妥至少一組 AI 服務 API 金鑰（請見 1.3 節）
  • USB 隨身碟為原始交付版本（未被複製或修改）
首次使用也可以先跳過 API 金鑰

如果還沒有任何 API 金鑰，可以先完成安裝，之後再透過 Web 設定頁面（Config.html）或主選單設定補填。推薦先申請免費的 Groq 金鑰快速體驗。

1.3　AI 服務金鑰申請指引

隨身黃仁蝦AI系統支援 10 家 AI 服務提供者。全部選用，您可依需求和預算選擇。

系統支援四種通訊頻道整合，讓 AI 助手可以透過您慣用的通訊軟體回應訊息。全部選用，不需要的可以略過。

第二章　安裝教學（Windows）

安裝過程極為快速，不需要管理員權限，全程約 1 分鐘即可完成。

插入 USB → 雙擊安裝檔 → 等待複製 → 完成!

2.1　插入 USB 並找到啟動檔

1 將隨身黃仁蝦AI USB 插入電腦

請將 USB 直接插入電腦主機板上的 USB 孔。避免使用 USB Hub 或延長線，以確保穩定的連線。

2 開啟 USB 磁碟機

開啟「檔案總管」，在左側找到 USB 磁碟機。您會看到以下檔案：

📁
【截圖預留】USB 根目錄畫面

應可看到「一鍵安裝」和「點這個啟動」兩個 .bat 檔案，以及 Welcome.html 歡迎頁

3 先看歡迎頁面（建議）

建議先雙擊 Welcome.html，瀏覽器會開啟快速上手指南，幫助您了解系統功能和 AI 模型選擇。

注意

如果 Windows 出現「Windows 已保護您的電腦」的 SmartScreen 警告，請點擊「其他資訊」>「仍要執行」。這是正常的安全提示，並非病毒。

2.2　執行一鍵安裝

找到 一鍵安裝-隨身黃仁蝦AI-Windows版.bat 並雙擊執行。

💻
【截圖預留】PowerShell 安裝畫面

顯示「隨身黃仁蝦AI系統 本機安裝」標題

安裝程式會自動執行以下步驟：

  1. 檢查 Node.js 和 OpenClaw 引擎
  2. 將系統檔案複製到本機（%LOCALAPPDATA%\隨身黃仁蝦AI系統-本機版）
  3. 產生 OpenClaw 設定檔
  4. 建立桌面捷徑
不需要管理員權限！

v2.0 的安裝完全在使用者目錄內完成，不會出現 UAC 權限確認對話框，也不需要 IT 管理員協助。公司電腦也能輕鬆安裝。

2.3　安裝完成確認

安裝成功後，您會看到以下訊息：

安裝完成。 本機安裝位置：C:\Users\您的帳號\AppData\Local\隨身黃仁蝦AI系統-本機版 之後請插入 USB，再雙擊「點這個啟動-隨身黃仁蝦AI-Windows.bat」。

同時，桌面會出現一個新的捷徑檔案：

🖥
【截圖預留】桌面顯示「黃仁蝦AI-本機版.bat」捷徑
安裝完成！

恭喜！隨身黃仁蝦AI系統已成功安裝到您的電腦。接下來請繼續第三章進行首次設定。

項目說明
本機安裝位置%LOCALAPPDATA%\隨身黃仁蝦AI系統-本機版
桌面捷徑黃仁蝦AI-本機版.bat
AI 引擎OpenClaw（Node.js 原生，無需 WSL）
安裝大小約 100 MB

2.4　Mac 安裝教學

v2.0 新增 Mac 支援！安裝流程與 Windows 類似，同樣快速簡單。

1 插入 USB 並開啟

插入 USB 後，在 Finder 中找到 USB 磁碟機。

2 雙擊「Mac-安裝.command」

系統會自動偵測您的 Mac 是 Apple Silicon（M1/M2/M3/M4）還是 Intel，選擇對應的 Node.js 版本。

安裝位置：~/.lobster-ai/

3 完成！

安裝完成後，桌面會出現「黃仁蝦AI-本機版.command」捷徑。之後啟動只需要雙擊 USB 上的 Mac-啟動.command。

macOS 安全性提示

首次執行 .command 檔案時，macOS 可能會顯示「無法打開，因為它來自不明的開發者」。 請到「系統設定 > 隱私與安全 > 安全性」中點選「仍要打開」即可。系統會自動處理 macOS 的隔離標記。

第三章　首次使用設定

安裝完成後的第一次啟動，系統會引導您完成設定精靈。設定完成後，日後啟動就不需要再重新設定。

3.1　啟動系統與 Email 驗證

1 確認 USB 已插入，雙擊啟動檔

Windows：雙擊 USB 上的 點這個啟動-隨身黃仁蝦AI-Windows.bat（或桌面捷徑）

Mac：雙擊 USB 上的 Mac-啟動.command

2 輸入 Email 進行驗證

系統會顯示歡迎畫面，並要求您輸入購買時綁定的 Email 地址。

🦐
【截圖預留】OpenClaw ASCII Art 歡迎畫面與 Email 輸入
重要

Email 必須與購買時綁定的完全一致。輸入錯誤將無法啟動系統。如果忘記綁定的 Email，請聯絡賣家確認。

3.2　設定精靈：終極密碼

系統偵測到是首次使用，會自動進入設定精靈。第一步是設定「終極密碼」（Master Password）。

🔒
【截圖預留】設定精靈 — 終極密碼設定畫面
注意

請務必牢記您的終極密碼！日後修改 API 金鑰、通訊頻道等重要設定時，都需要輸入此密碼。 此密碼也用於加密您的 API 金鑰，確保安全性。建議記在安全的地方。

3.3　設定精靈：AI 服務金鑰

接下來系統會依序詢問以下 API 金鑰（全部選填，直接按 Enter 略過）：

#欄位推薦提示

2DeepSeek API Key推薦最便宜的付費方案
3Qwen API Key選填低成本付費方案
4OpenAI API Key選填GPT 系列
5Anthropic API Key選填Claude 系列
6Gemini API Key選填Google AI

不需要的欄位直接按 Enter 略過即可。日後隨時可以從 Web 設定頁面（Config.html） 或主選單 [5] 設定功能補填或修改。 API 金鑰會以 AES-256 加密儲存，確保安全。

3.4　設定精靈：通訊頻道

設定 AI 助手要透過哪些通訊軟體與您互動：

💬
【截圖預留】設定精靈 — 通訊頻道設定畫面

系統會先詢問「是否要設定通訊頻道？」

如果選擇「是」，系統會依序詢問各頻道的設定資訊：

4.3　啟動 OpenClaw AI 服務

啟動服務

在主選單輸入 1 並按 Enter。系統會：

  1. 驗證 USB 授權（Device Token + HMAC 簽章）
  2. 檢查授權狀態（永久/訂閱/試用）
  3. 從 config.env 同步產生 OpenClaw 設定檔
  4. 掃描可用 port（18789-18799）
  5. 啟動 OpenClaw Gateway
  6. 自動開啟瀏覽器到 AI 介面
▶
【截圖預留】OpenClaw Web UI 介面

瀏覽器會自動開啟 http://127.0.0.1:18789

查看狀態

在主選單輸入 3 查看目前服務狀態，包括：

  • Node.js 版本
  • OpenClaw 安裝狀態
  • Gateway 運行狀態（含 port）
  • 已設定的 API Key 數量
  • 安裝模式（USB / 本機）
  • 授權狀態（永久 / 訂閱剩餘天數 / 試用期）
  • 已安裝技能數量

停止服務

在主選單輸入 9 離開系統，所有背景服務（包括 OpenClaw Gateway 和 Config Server）會自動清理。

4.7　USB 安全保護說明

隨身黃仁蝦AI系統 v2.0 內建多層安全保護機制：

機制說明
Email 綁定每次啟動需驗證購買時綁定的 Email
Device Token每支 USB 有唯一識別碼，複製的 USB 無法使用

第六章　常見問題與疑難排解

6.1　安裝階段問題

問題可能原因解決方法
雙擊 .bat 檔沒有反應 被防毒軟體攔截 暫時關閉防毒軟體的即時防護，或將 USB 路徑加入白名單。
SmartScreen 警告 Windows 安全機制 點擊「其他資訊」>「仍要執行」。這是正常提示，並非病毒。
安裝過程中斷 USB 讀取不穩 確認 USB 穩固插入（避免 Hub），重新執行安裝檔。已完成的步驟會自動跳過。
Mac 顯示「無法打開」 macOS Gatekeeper 系統設定 > 隱私與安全 > 安全性 > 點選「仍要打開」。
中文顯示亂碼 未透過 .bat 檔啟動 .bat 檔會自動設定 UTF-8 編碼。請務必從 .bat 檔啟動，不要直接執行 .ps1 檔。

6.2　啟動與使用問題

問題可能原因解決方法
「帳號不符，無法啟動」 Email 輸入錯誤 請確認輸入的 Email 與購買時綁定的完全一致（注意大小寫）。如已忘記，請聯絡賣家。
「未偵測到授權 USB」 USB 未插入或 Device Token 不符 確認已插入原始 USB（非複製品）。如果是從桌面捷徑啟動，仍需要插入 USB。
「授權已到期」 訂閱或試用期到期 聯絡賣家續費或升級為永久授權。
「Device Token 完整性驗證失敗」 config.env 被手動修改 請勿手動編輯 config.env 中的 DEVICE_TOKEN 欄位。如遇此問題，聯絡賣家重新綁定。
OpenClaw 啟動失敗 Port 被占用 系統會自動掃描 18789-18799 找空閒 port。如果全部被占用，請關閉其他佔用的程式。
瀏覽器沒有自動開啟 瀏覽器路徑問題 手動開啟瀏覽器，輸入終端機中顯示的網址（如 http://127.0.0.1:18789）。
每 10 分鐘跳出警告 USB 被拔除或接觸不良 確認 USB 已穩固插入。如使用 Hub，改為直接插入電腦 USB 孔。

6.3　通訊頻道問題

問題可能原因解決方法
LINE Bot 沒有回應 API Key 錯誤或 Webhook 未設定 確認 LINE Developers 後台的 Channel Access Token 正確，且 Webhook URL 已設定。
Telegram Bot 無回應 Bot Token 錯誤或 User ID 未授權 從 @BotFather 確認 Token，從 @userinfobot 確認 User ID，並在設定中更新。
Discord Bot 離線 Bot Token 失效或未邀請到伺服器 從 Discord Developer Portal 重新產生 Token，並確認 Bot 已加入目標伺服器。

6.4　效能與穩定性問題

問題可能原因解決方法
AI 回應很慢 AI 服務端問題 嘗試切換到不同的 AI 提供者。Groq 回應速度通常最快。
Dashboard 打不開 Config Server 未啟動或 Port 被占用 從主選單 [7] 重新啟動。或手動開啟瀏覽器到 http://127.0.0.1:18788。
USB 讀取速度慢 使用 USB 2.0 或透過 Hub 改用 USB 3.0 孔（通常為藍色），避免使用 Hub 或延長線。
