# Git 操作

## 情境
使用 Git 版本控制時遇到問題，例如合併衝突、誤刪分支、需要回復到某個版本等。讓 AI 根據你的狀況提供正確的 Git 指令和操作步驟。

## 示範 Prompt
> 我在 Git 操作中遇到以下狀況，請幫我解決：
>
> 我在 feature/login 分支開發完成後，要合併回 main 分支，但出現了大量合併衝突。我想要：
> 1. 先保留目前的變更，暫時回到乾淨狀態
> 2. 把 main 分支最新的變更先 rebase 到我的分支上
> 3. 逐一解決衝突後完成合併
> 4. 如果 rebase 搞砸了，能回到 rebase 之前的狀態
>
> 請給我完整的指令步驟，並說明每步在做什麼。

## 預期效果
AI 會產出：
- 完整的 Git 指令流程，包含 stash、rebase、conflict resolve、abort 等操作步驟
- 每個指令的中文說明，解釋執行後會發生什麼事
- 安全操作建議，例如 rebase 前先建立備份分支、使用 reflog 救回誤操作等
