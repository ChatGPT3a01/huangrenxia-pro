# 資料庫設計

## 情境
要開發新系統或新功能時，需要設計資料庫架構。描述你的業務需求，讓 AI 幫你規劃資料表結構、欄位定義和關聯關係，避免後續反覆修改。

## 示範 Prompt
> 我要設計一個線上訂餐系統的資料庫，業務需求如下：
>
> 1. 有多家餐廳，每家餐廳有自己的菜單
> 2. 菜單分類別（主餐、飲料、甜點等），每道菜有名稱、價格、描述、圖片
> 3. 客戶可以註冊帳號、儲存多個外送地址
> 4. 客戶下訂單時可選擇不同餐廳的餐點，每筆訂單要記錄外送地址和備註
> 5. 訂單有狀態流程：待確認 → 製作中 → 外送中 → 已送達
> 6. 需要記錄每道菜的評分和評論
>
> 請用 MySQL 語法產生建表 SQL，並畫出 ER 關係說明。

## 預期效果
AI 會產出：
- 完整的 CREATE TABLE SQL 語句，包含主鍵、外鍵、索引和適當的資料型別
- 資料表關聯說明（文字版 ER 圖），清楚標示一對多、多對多關係
- 設計考量說明：為何這樣分表、索引策略、未來擴充性建議
